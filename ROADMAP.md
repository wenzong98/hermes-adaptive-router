## 近期可拓展功能（P1 — 高价值，低复杂度）

### 1. 更多 Provider 支持
- [ ] **Google Search** (SerpAPI / Programmable Search Engine)
- [ ] **Bing Search** (Azure Cognitive Search)
- [ ] **DuckDuckGo** (无需 API key)
- [ ] **Brave Search**
- [ ] **Perplexity API** (自带 AI 摘要)

**工作量**: 每个 Provider ~50 行 (关键词信号 + fallback 逻辑)
**价值**: 用户可根据查询语言/领域选择最优后端

### 2. 查询意图增强
- [ ] **代码搜索信号**: `github`, `stackoverflow`, `代码`, `源码`
- [ ] **图片搜索信号**: `图片`, `image`, `photo`, `截图`
- [ ] **视频搜索信号**: `视频`, `youtube`, `教程`
- [ ] **文档搜索信号**: `文档`, `docs`, `API reference`
- [ ] **购物搜索信号**: `购买`, `buy`, `amazon`, `淘宝`

**工作量**: 每个类别 ~10 个关键词
**价值**: 更细粒度的路由决策

### 3. 语言检测自动路由
- [ ] 集成 `langdetect` 或 `fasttext` 检测查询语言
- [ ] 中文 → 优先 MMX，英文 → 优先 Tavily/Exa
- [ ] 多语言混合查询处理

**工作量**: ~100 行
**价值**: 消除对关键词的依赖，更准确

### 4. 路由决策缓存
- [ ] LRU 缓存最近 N 个查询的路由决策
- [ ] 缓存 key = hash(查询文本归一化)
- [ ] TTL 过期（如 1 小时）

**工作量**: ~50 行
**价值**: 高频重复查询（如"今天天气"）避免重复计算

### 5. A/B 测试框架
- [ ] 支持两种路由策略并行运行
- [ ] 自动对比准确率（人工标注 vs 路由结果）
- [ ] 生成对比报告

**工作量**: ~200 行
**价值**: 验证新策略是否优于旧策略

## 中期可拓展功能（P2 — 中等价值，中等复杂度）

### 6. 机器学习分类器（轻量）
- [ ] 基于历史数据训练简单分类器（如 scikit-learn 的 SVM/RandomForest）
- [ ] 特征: 词频、关键词命中、长度、语言
- [ ] 可切换: 确定性规则 vs ML 模型
- [ ] 模型定期从 JSONL 历史数据重训练

**工作量**: ~300 行
**价值**: 随着时间推移，分类准确率自动提升

### 7. 实时性能监控
- [ ] 路由延迟 histogram（P50/P95/P99）
- [ ] 按 provider 统计成功率/失败率
- [ ] 健康检查: provider 可用性探测
- [ ] Prometheus / StatsD 导出

**工作量**: ~200 行
**价值**: 生产环境可观测性

### 8. 动态阈值调整
- [ ] 根据历史准确率自动调整 `simple_max_words`
- [ ] 根据用户反馈（👍/👎）调整关键词权重
- [ ] 在线学习: 每次用户纠正路由决策，更新内部权重

**工作量**: ~300 行
**价值**: 自适应优化，无需人工调参

### 9. 多轮对话上下文感知
- [ ] 记住前几轮的路由决策
- [ ] 同一话题后续查询继承上轮 datasource
- [ ] 话题切换检测（关键词突变）

**工作量**: ~200 行
**价值**: 减少同一话题内的反复路由切换

### 10. 查询重写/扩展
- [ ] 自动补全模糊查询（如"gpt" → "GPT-5 最新价格"）
- [ ] 查询扩展: 添加同义词提升搜索召回
- [ ] 查询消歧: 多义词检测（如"apple" → 公司 vs 水果）

**工作量**: ~400 行
**价值**: 提升搜索质量

## 远期可拓展功能（P3 — 高价值，高复杂度）

### 11. 端到端评估框架
- [ ] 构建 golden dataset（人工标注的查询→路由对）
- [ ] 自动化评估: precision/recall/F1 per class
- [ ] 回归测试: 每次代码变更自动跑评估
- [ ] 基准对比: 与 LLM-based 分类器（如 GPT-4）对比成本和准确率

**工作量**: ~500 行 + 数据标注
**价值**: 量化路由质量，指导优化方向

### 12. 联邦路由（跨实例）
- [ ] 多个 Hermes 实例共享路由决策
- [ ] 分布式缓存（Redis）
- [ ] 全局统计聚合

**工作量**: ~400 行
**价值**: 大规模部署场景

### 13. 语义路由（Embedding-based）
- [ ] 预计算查询 embedding
- [ ] 与已知查询模式做余弦相似度匹配
- [ ] fallback 到关键词规则

**工作量**: ~300 行 + 模型依赖
**价值**: 捕获关键词无法表达的语义模式

### 14. 对话状态机
- [ ] 定义对话阶段（开场 → 信息收集 → 深度搜索 → 总结）
- [ ] 每阶段有不同的路由策略
- [ ] 状态转换由用户行为和工具结果驱动

**工作量**: ~600 行
**价值**: 更智能的对话流程控制
